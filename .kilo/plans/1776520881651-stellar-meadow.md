### Phase 4: Execution Order (Safety Guidelines)

1. **Start with Layout fixes** - Most isolated, easiest to test
2. **Then CSS fixes** - Self-contained parser changes
3. **Then Rendering fixes** - Defensive changes

**Safety Rules**:
1. Run `dotnet test` before and after each change
2. Check specific project tests match affected area
3. If tests fail, investigate first - don't assume the fix broke it
4. Make one change at a time, test, then proceed

**Rollback**:
- Each fix is isolated - can revert specific files
- Tests provide verification
